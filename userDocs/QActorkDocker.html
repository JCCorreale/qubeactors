Deploy QActork using Docker

Deploying a Single QActork Context

  1. Create a QActork executable model with the following code:
    System hellodocker

Context hellocontext ip [host="localhost" port=8080]

QActor hello context hellocontext
{
	State s0 initial {
		
		println("Hello")
	}
}
    2. Note: If the run fails with Error: Could not find or load main class it may mean that the IDE can't find the *Kt Kotlin file related to the main class. To fix this, edit the corresponding Run Configuration in the Dependencies tab (or the Classpath tab, depending on the Eclipse version), choose Advanced and enter the folder containing the kotlin generated .class files (for example: build/classes/kotlin/main).
    Note: When changing libraries in the build.gradle file, it may happen that the runtime dependencies are not correctly updated. In that case, just delete the current run configuration and repeat the above procedure.
  2. To deploy our QActork context on Docker, we need to create deployable artifacts for that context. To do that, edit the context-specific build file (for example: build_hellocontext.gradle) by adding the following code:
    Note:  The application plugin must be added to the plugins section of the Gradle build file.
     
    mainClassName = 'it.unibo.hellocontext./MainHellocontextKt'

jar {
    println("executing jar")
    from sourceSets.main.allSource
    manifest {
        attributes 'Main-Class': "$mainClassName"
    }
}
  3. Build the deployable artifacts: 
    gradle -b build_hellocontext.gradle build
  4. Create a file named Dockerfile in the project root, with the following content (where qubeactors-sandbox is the name of the project): 
    FROM openjdk:12.0.2
EXPOSE 8080
ADD ./build/distributions/*.tar /
ADD ./*.pl /
RUN ls /qubeactors-sandbox-1.0/bin
ENTRYPOINT ["bash", "/qubeactors-sandbox-1.0/bin/qubeactors-sandbox"]
  5. Build the docker image with the following command: 
    docker build --rm -t hello-qactor .
  6. Run the docker image: 
    docker run --rm -p8080:8080 hello-qactor
